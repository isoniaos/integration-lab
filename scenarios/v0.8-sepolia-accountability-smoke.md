# v0.8 Sepolia Accountability Smoke Scenario

## Purpose

Validate that a real Sepolia IsoniaOS governance decision can be presented with
public evidence, external context, Control Plane accountability output, and App
Core presentation output.

This scenario is a smoke test and presentation baseline. It is not a production
provider integration certification.

## Preconditions

- A live Sepolia RPC endpoint.
- Sepolia test ETH for the operator account.
- An IsoniaOS governance deployment on Sepolia, or approval to deploy one.
- Control Plane pointed at the selected Sepolia deployment.
- App Core configured to read from the selected Control Plane instance.
- Access to any external provider accounts needed for optional evidence links.

## Steps

1. Deploy or select an Isonia governance deployment on Sepolia.
2. Create, approve, and execute one governance decision.
3. Capture the transaction hash and block explorer proof for each relevant
   transaction.
4. Attach or link Snapshot testnet proposal, discussion, or signaling context
   when available.
5. Attach or link Safe transaction proof when available.
6. Optionally test Tally or OpenZeppelin Governor compatibility if a compatible
   Governor deployment exists.
7. Optionally record Agora notes when relevant to the governance scenario.
8. Verify Control Plane archive and accountability output for the selected
   organization and proposal.
9. Verify App Core public page presentation for the selected organization and
   proposal.

## Required Evidence

- Sepolia deployment manifest or selected deployment record.
- Proposal identifier and organization identifier.
- Transaction hashes for create, approval, queue, execute, cancel, veto, or
  other lifecycle actions exercised by the smoke test.
- Block explorer URLs for transaction proof.
- Control Plane route or accountability response captures.
- App Core public page URL or screenshot reference.

## Optional Evidence

- Snapshot testnet proposal or discussion link.
- Safe transaction service link or explorer transaction link.
- Tally Governor compatibility notes.
- Agora research or linking notes.
- GitHub issue, pull request, or Discourse discussion links used as context.

## Pass Criteria

- The selected governance decision is visible on Sepolia.
- Control Plane output reflects the proposal lifecycle and accountability route.
- App Core can present the organization and proposal in a public-facing form.
- External provider records are labeled with the correct trust boundary.

## Failure Notes

Record failures as observations rather than silently adjusting the scenario.
Examples include missing provider support, unsupported contract surfaces,
missing Control Plane projections, wallet or RPC failures, and App Core
presentation gaps.
