# v0.8 Sepolia Managed Execution Smoke Scenario

## Purpose

Prepare a live Sepolia field test for org-scoped managed execution without
placing provider assumptions, demo fixtures, or lab-only state into core
repositories.

This scenario proves the observable path:

```txt
Isonia proposal approved
-> final target, value, actionSelector, and dataHash modeled
-> final target, selector, and value permission rules enabled
-> optional org-scoped executor forwards execution
-> canonical ProposalExecuted receipt emitted by GovProposals
-> Control Plane indexes direct or managed execution proof
-> App Core displays archive, accountability, and managed execution evidence
```

This is a field-test smoke kit, not a production, audit, SaaS, provider
completeness, or public beta claim.

## Scope

Use this scenario for:

- live Sepolia operator preparation;
- pinned Sepolia fork replay planning;
- external evidence capture planning;
- Control Plane and App Core read-model verification;
- provider boundary notes for Snapshot, Safe, Tally, Agora, block explorers,
  GitHub, and Discourse.

Do not use this scenario as:

- a runtime configuration source for Control Plane or App Core;
- proof that external provider records are IsoniaOS authority;
- proof that a provider integration is complete;
- a replacement for protocol deployment records or release notes.

## Required Inputs

- A selected Sepolia IsoniaOS v0.8 deployment with managed execution support.
- Sepolia chain id `11155111`.
- `govCore` and `govProposals` addresses.
- Optional org-scoped executor address.
- Organization id.
- Control Plane protocol profile and deployment capability metadata.
- One final execution target, selector, value, and data hash.
- Target, selector, and value permission rule evidence.
- Canonical `ProposalExecuted` receipt evidence after execution.

The example manifest is:

```txt
sepolia/managed-execution-manifest.example.json
```

## Smoke Flow

1. Copy the example manifest into a run-specific file outside the committed
   example path.
2. Replace all example-only addresses, hashes, ids, block numbers, and URLs
   with run-specific Sepolia values.
3. Confirm the organization has the expected org-scoped executor configured, or
   record that the scenario is using direct execution.
4. Confirm the final target permission rule is enabled for the organization.
5. Confirm the final selector permission rule is enabled for the target.
6. Confirm the value limit permits the planned execution value.
7. Create or select a proposal whose modeled final action contains:
   `targetAddress`, `value`, `actionSelector`, `dataHash`, and `metadataUri`.
8. Move the proposal through the required Isonia approval route.
9. Execute either directly through `GovProposals` or through the org-scoped
   executor.
10. Capture the canonical `ProposalExecuted` receipt emitted by `GovProposals`.
11. Run Control Plane indexing against the selected Sepolia deployment.
12. Verify App Core displays public archive, accountability, and managed
   execution receipt evidence for the proposal.
13. Attach external records only as evidence or context, with source disclosure.

## Primary Target Case: Ownable-Style Managed Target

The primary smoke target is an Ownable-style target where the current owner is
expected to be either `GovProposals` for direct execution or the org-scoped
executor for managed execution.

Required observations:

- the target owner before execution;
- the target owner or state after execution when relevant;
- the Isonia proposal identity fields;
- the permission rules for target, selector, and value;
- the canonical execution receipt;
- whether execution used direct or managed forwarding.

Target ownership is target-contract state. IsoniaOS authority still comes from
modeled protocol state, permission rules, proposal lifecycle state, and emitted
canonical receipts.

## Optional Future Evidence: AccessControl-Style Role Handoff

The optional future evidence lane is an AccessControl-style target where the
org-scoped executor or protocol-controlled address receives a role before the
final action is attempted.

Record this as future or compatibility evidence until the exact target
contract, role id, admin path, and execution route are validated on Sepolia.
Do not claim broad AccessControl support from one field test.

## Required Evidence

- Run-specific managed execution manifest.
- Sepolia deployment or selected-contract record.
- Organization id.
- `govCore`, `govProposals`, and optional org executor addresses.
- Proposal id and proposal identity fields.
- Target, selector, and value permission rule evidence.
- Approval route completion evidence.
- Execution transaction hash and block number.
- Canonical `ProposalExecuted` receipt fields:
  `proposalId`, `executorAddress`, `targetAddress`, `value`,
  `actionSelector`, `dataHash`, `managedExecutorAddress`, `txHash`, and
  `blockNumber`.
- Control Plane archive, accountability, and execution receipt output.
- App Core public archive/accountability/managed execution display evidence.

## External Evidence Boundaries

- Snapshot testnet space or proposal links are signaling, discussion, or
  context evidence.
- Safe Sepolia transaction links are workflow or transaction proof evidence.
- Tally Governor pages are compatibility research unless a compatible Governor
  path is explicitly validated and scoped.
- Agora links are research or linking notes unless a compatible path is
  explicitly validated and scoped.
- Block explorer links prove transaction, block, address, or log visibility;
  explorer UI labels are not governance authority.
- GitHub and Discourse links are implementation, release, discussion, or
  decision context.

External records must not be marked authoritative by default.

## Pass Criteria

- The proposal approval route completes under the selected org and policy.
- The final target, value, action selector, and data hash match the proposal
  identity fields recorded in the manifest.
- Target, selector, and value permission rules are present and enabled.
- The execution path is clearly labeled as direct or managed.
- `GovProposals` emits the canonical `ProposalExecuted` receipt.
- Control Plane indexes the execution proof and managed executor details.
- App Core renders public archive, accountability, and managed execution
  evidence with source disclosure.

## Failure Notes

Record failures as observations. Do not silently change the scenario to make a
run pass.

Examples:

- missing or disabled target rule;
- selector mismatch;
- value limit mismatch;
- missing org executor;
- execution forwarded by an unexpected address;
- missing canonical receipt projection;
- App Core display gap;
- provider UI/API unable to show relevant Sepolia state;
- fork-only transaction mistaken for live provider evidence.
